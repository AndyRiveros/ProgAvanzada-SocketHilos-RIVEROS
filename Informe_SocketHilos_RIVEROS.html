1. Descripcion del Problema

Esta practica es la segunda parte del trabajo de Sockets en Java, tomando como base la "Practica de Socket base - 2026" (ya aprobada). El objetivo es extender el servidor para que pueda atender multiples clientes de forma concurrente mediante hilos (Threads), implementando un sistema de chat completo con comandos.

Diferencia clave con la practica anterior: En el socket base, el servidor solo podia atender un cliente a la vez (bloqueante). En esta version, cada cliente nuevo genera un hilo independiente, permitiendo multiples conexiones simultaneas sin bloqueos.

2. Arquitectura del Sistema

  [Cliente 1]  ---- TCP:5000 ----                                   +--- [Servidor] --- [ManejadorCliente - Hilo 1]
  [Cliente 2]  ---- TCP:5000 ----/          |
                                             +---------- [ManejadorCliente - Hilo 2]
  [Cliente N]  ---- TCP:5000 ------------- ...

El servidor escucha en el puerto 5000. Por cada conexion entrante, instancia un ManejadorCliente y lo ejecuta en un nuevo Thread. El mapa de clientes es compartido y protegido con ConcurrentHashMap y metodos synchronized.

ClaseResponsabilidadPatron
Servidor.javaAcepta conexiones, gestiona mapa de clientes, logSingleton del estado global
ManejadorCliente.javaAtiende a un cliente, procesa comandosRunnable / Hilo por cliente
Cliente.javaEnvia/recibe mensajes, hilo receptorHilo concurrente (daemon)

import java.io.*;
import java.net.*;
import java.text.SimpleDateFormat;
import java.util.*;
import java.util.concurrent.ConcurrentHashMap;

public class Servidor {
    private static ConcurrentHashMap<String, ManejadorCliente> clientes = new ConcurrentHashMap<>();

    public static void main(String[] args) {
        int puerto = 5000;
        System.out.println("Servidor de Chat con Hilos - RIVEROS");
        System.out.println("Escuchando en puerto " + puerto + "...");

        try (ServerSocket serverSocket = new ServerSocket(puerto)) {
            while (true) {
                Socket socket = serverSocket.accept();
                log("Nueva conexion desde: " + socket.getInetAddress());
                new Thread(new ManejadorCliente(socket)).start();
            }
        } catch (IOException e) {
            System.err.println("[ERROR] " + e.getMessage());
        }
    }

6. Caracteristicas de Concurrencia

MecanismoUso en el proyecto
Thread + RunnableUn hilo por cada cliente conectado (ManejadorCliente)
ConcurrentHashMapMapa de clientes thread-safe sin bloqueos innecesarios
synchronizedMetodos de acceso al mapa de clientes para evitar race conditions
Hilo daemonHilo receptor en el Cliente (muere con el proceso principal)
Ventaja del modelo: Mientras un cliente espera input del usuario, los demas clientes son atendidos normalmente por sus propios hilos. No hay bloqueos cruzados entre clientes.

7. Conclusiones

Esta practica extendio exitosamente el servidor de socket base para soportar multiples clientes concurrentes mediante el uso de hilos Java (Threads). Se implementaron todos los comandos requeridos por el enunciado: mensajeria entre clientes (individual y broadcast), consulta de fecha/hora, calculo de expresiones matematicas, listado de clientes conectados y un sistema de log completo en la consola del servidor.

La solucion utiliza ConcurrentHashMap para el manejo thread-safe del registro de clientes y metodos synchronized para garantizar consistencia en las operaciones de escritura sobre el estado compartido.

Relacion con la practica anterior: La "Practica de Socket base - 2026" (aprobada, entregada el 11/04/2026) implemento el servidor simple de un solo cliente. Esta practica toma ese codigo como base y lo refactoriza completamente para soporte multi-cliente con hilos, manteniendo la misma logica de comunicacion TCP y el mismo estilo de codigo.
